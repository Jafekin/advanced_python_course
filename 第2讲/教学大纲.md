# 第2讲：数据高阶可视化 × AI 辅助表达 — 教学大纲

## 一、本讲教学目标

### 知识目标
1. 掌握 Matplotlib 进阶绑定图表类型选择策略（图表决策树），能根据数据特征选择合适图表
2. 掌握 Seaborn 高阶统计图表（热力图、小提琴图、分面图等）的原理与适用场景
3. 理解科研论文图表的排版规范（Nature 标准、300dpi、色盲友好配色）
4. 了解 AI 辅助可视化的三轮迭代法及工作流程

### 能力目标
1. 能用 Matplotlib 完成多子图、分组柱状图、折线图美化等进阶操作
2. 能用 Seaborn 进行多维数据探索性可视化分析
3. 能独立产出符合发表标准的科研图表（误差棒、置信区间、双轴图、显著性标注）
4. 能与 AI 协作完成"从粗图到发表级"的三轮迭代优化

### 态度目标
1. 建立"数据可视化即沟通"的理念，重视图表的表达准确性与美学
2. 培养对 AI 生成方案的批判性评估能力，而非盲从
3. 形成"先想清楚再画图"的科研习惯

## 三、两条主线融合设计

### 知识主线（数据可视化进阶）
```
Matplotlib 基础回顾 → 图表类型决策 → Seaborn 统计图表 → 科研发表标准 → 综合应用
```

### 工具主线（Vibe Coding — AI 作为设计顾问）
```
AI 方案推荐 → AI 方案对比 → Diff 审查 → 三轮迭代（氛围→约束→Inline Edit）→ AI 视觉审查
```

### 融合节点

| 知识点 | AI 角色 | 协作方式 |
|--------|---------|---------|
| 图表决策树 | 设计顾问 | 描述数据 → AI 推荐图表类型 + 理由 |
| Seaborn 多图对比 | 评审员 | AI 对比多种图表方案，给出表达力评分 |
| 科研论文排版 | 排版助手 | AI 检查 dpi/字号/配色/轴标签是否合规 |
| 从粗图到精图 | 迭代伙伴 | 三轮 Prompt 逐步收紧约束，Inline Edit 微调细节 |

**本讲 AI 角色定位 = 设计顾问**：AI 不只是生成代码，更重要的是提供可视化方案建议、视觉审查反馈、配色合规检查，学生需要判断和选择。

## 四、重点难点与应对策略

### 重点
1. 图表类型决策能力（不同数据结构 → 最佳图表映射）
2. 科研论文图表的完整规范（尺寸/dpi/配色/标注/字体）
3. AI 三轮迭代的 Prompt 设计策略

### 难点
1. **rcParams 全局设置与局部样式冲突**
   - 应对：先演示冲突案例，再讲 `with plt.style.context()` 上下文管理
2. **统计标注（p 值/置信区间）的正确使用**
   - 应对：从"什么时候需要标注"讲起，配合 scipy.stats 实操
3. **AI 生成图表的批判性评估**
   - 应对：展示 AI 犯错案例（配色不色盲友好/误用双轴图），培养审查意识

### 应对策略
- 每个 Part 先给出"坏图 vs 好图"对比，建立直觉
- 科研标准部分提供 checklist 供学生自查
- AI 协作部分强调"先理解再使用"，避免黑盒依赖
