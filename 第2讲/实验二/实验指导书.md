### 任务1：Matplotlib 基础可视化（20分）

**目标**：用 Matplotlib 对全球气温年度数据进行基础可视化分析。

#### 要求

**1.1 趋势折线图（正负距平分色填充）（8分）**

- 筛选 GISTEMP 数据源的年度数据（Month == 0）
- 画出 1880-2024 年度气温距平折线图
- 使用 `fill_between`：正距平填充红色（alpha=0.3），负距平填充蓝色（alpha=0.3）
- 添加零线（y=0 水平虚线）
- 中文标题："全球年度气温距平趋势（1880-2024）"
- X 轴标签："年份"，Y 轴标签："气温距平 (℃)"

**1.2 年代均值柱状图（6分）**

- 按年代（decade：1880s/1890s/.../2020s）分组计算均值
- 画柱状图，柱子颜色按均值正负分色
- 每根柱子顶部/底部标注数值（保留2位小数）
- 中文标题："各年代全球平均气温距平"

**1.3 四季趋势多子图（6分）**

- 将月份划分为四季：DJF(12,1,2) / MAM(3,4,5) / JJA(6,7,8) / SON(9,10,11)
- 用 2×2 子图分别展示每个季节的年度平均距平趋势折线
- 每个子图标题为季节名（如"冬季 DJF"）
- 子图间距适当，不重叠

**通用要求**：
- 所有图表保存为 PNG，dpi=300
- 使用 `plt.rcParams` 设置中文字体（SimHei 或系统可用中文字体）
- 图片保存至 `output/` 目录

### 任务2：Seaborn 高阶探索（25分）

**目标**：利用 Seaborn 的高阶统计图表从多维度探索气温数据的分布与模式。

#### 要求

**2.1 热力图：月份 × 年代的气温距平矩阵（10分）**

- 数据处理：筛选 GISTEMP 月度数据（Month 1-12），按年代和月份分组求均值
- 构建透视表：行=月份(1-12)，列=年代(1880s-2020s)
- 用 `sns.heatmap` 绘制，使用 `RdBu_r` 色图（红暖蓝冷）
- 添加数值标注（保留1位小数），设置合理的 vmin/vmax 使色彩对称
- 中文标题："各月份 × 年代平均气温距平热力图"
- Y 轴标签用月份名（1月/2月/.../12月）

**2.2 小提琴图：两个时期的月度气温分布对比（8分）**

- 将数据分为两个时期：早期（1880-1960）和近期（1961-2024）
- 用 `sns.violinplot` 对比两个时期的月度气温距平分布
- 设置 `split=True` 展示左右对比（若数据支持）或并排对比
- 添加均值标记点
- 中文标题："早期(1880-1960) vs 近期(1961-2024) 月度气温分布"

**2.3 FacetGrid 分面图：四季年度趋势（7分）**

- 将数据按四季分组（DJF/MAM/JJA/SON）
- 使用 `sns.FacetGrid` 创建分面图，每面展示该季节的年度均值趋势散点+回归线
- 用 `sns.regplot` 或 `lmplot` 拟合趋势
- 每个分面标题标注季节名称和趋势斜率
- 统一 Y 轴范围以便跨季节对比

**通用要求**：
- 保存至 `output/` 目录，dpi=300
- 注意图表尺寸（figsize），确保标注清晰可读

### 任务3：科研论文级图表（30分）

**目标**：产出一张完全符合 Nature 期刊标准的科研图表。

#### 要求

**3.1 主图绘制（15分）**

- 绘制 1880-2024 年度气温距平折线图（GISTEMP 数据源）
- 叠加 10 年滑动平均曲线（粗线，深色）
- 用 `fill_between` 绘制 95% 置信区间带（基于 10 年窗口的标准误差 ×1.96）
- 标注历史最暖年份：2016 年和 2023 年（用箭头+文字标注）

**3.2 统计标注（8分）**

- 对"早期"（1880-1920）和"近期"（1990-2024）两个时段的年度距平进行独立样本 t 检验
- 在图上标注：
  - 两个时段的均值水平线（虚线）
  - 连接两段的方括号标注
  - p 值和显著性星号（如 `***p < 0.001`）
- 使用 `scipy.stats.ttest_ind` 完成检验

**3.3 发表级排版（7分）**

严格遵守以下规范：
- 图片宽度：3.5 inch（Nature 单栏标准）
- 分辨率：300 dpi
- 背景：白底，无网格线
- 字体：Arial，轴标签 8pt，标题 9pt
- 配色：色盲友好（Wong 2011 配色方案）
  - 主折线：`#0072B2`（蓝）
  - 滑动平均：`#D55E00`（橙红）
  - 置信区间：`#0072B2`（蓝，alpha=0.2）
  - 标注文字：`#000000`（黑）
- 坐标轴：左和底轴可见，右和顶轴隐藏（`sns.despine()`）
- 输出格式：同时保存 PDF 和 PNG（`output/fig_nature.pdf` + `output/fig_nature.png`）

### 任务4：AI 辅助三轮迭代（25分）

**目标**：体验 AI 辅助可视化的完整三轮迭代流程，培养 Prompt 设计能力。

#### 要求

**4.1 选择目标图表（2分）**

- 从任务1-3中选择一张图表作为目标
- 说明选择理由（建议选择任务3的主图，因其复杂度适合迭代优化）

**4.2 三轮迭代（18分，每轮6分）**

**Round 1 — 氛围提示（Vibe Prompt）**：
- 用自然语言描述你想要的图表效果（不超过3句话）
- 例如："帮我画一张全球气温变化的折线图，要有平滑趋势线和不确定性范围"
- 记录 AI 生成的代码和图表
- 截图保存

**Round 2 — 约束提示（Constraint Prompt）**：
- 在 Round 1 基础上追加具体约束条件
- 包括：精确的尺寸/dpi/配色/字体/标注要求
- 例如添加"宽度3.5inch，300dpi，Wong配色，Arial 8pt标签，标注2016和2023年"
- 记录 AI 修改后的代码和图表
- 截图保存

**Round 3 — Inline Edit 微调**：
- 使用 CodeBuddy 的 Inline Edit 功能对 Round 2 的代码进行局部微调
- 至少做 2 处微调（如调整标注位置、修改颜色透明度、调整图例位置等）
- 记录每处修改的 Prompt 和结果
- 截图保存

**4.3 对比分析（5分）**

- 将 Round 1 和 Round 3 的图表并排对比
- 从以下维度分析改进：
  - 信息传达清晰度
  - 视觉美观度
  - 科研规范符合度
  - 代码简洁度
- 总结三轮迭代的 Prompt 设计策略心得（100字以上）
