## 一、本讲教学目标

### 1.1 知识目标

1. 理解大模型部署的工程全景：从权重文件到 HTTP API 的完整链路
2. 掌握 Ollama 本地部署的完整操作流程（安装/拉取/服务/验证）
3. 理解量化原理（PTQ/QAT）及 GGUF K-Quants 格式选型
4. 了解 vLLM 的核心技术（PagedAttention / 连续批处理）
5. 理解 RAG 三段式架构（检索-增强-生成）和三代演进
6. 掌握 RAG 流水线六环节的原理和代码实现
7. 能根据场景选择合适的分块策略和嵌入模型

### 1.2 能力目标

1. 能在 CloudStudio 上独立部署 Ollama 并验证 API 可用
2. 能使用 markitdown 加载文档并实现递归字符分块
3. 能用 sentence-transformers + ChromaDB 构建向量库
4. 能从零实现完整 RAG 问答流水线，不依赖 LangChain
5. 能设计防幻觉 Prompt 模板，实现带引用的结构化回答

### 1.3 态度目标

1. 建立"部署是工程化的最后一公里"的认知
2. 理解 RAG 是大模型落地最实用的范式之一
3. 形成"每一步都可调试、可替换"的模块化工程思维

## 三、知识线与工具线融合设计

**【核心设计思路】** 本讲是"从调用到部署+检索"的关键跨越：第4-5讲学生已经会调用和微调大模型，本讲让他们亲手把模型跑起来，并用 RAG 解决幻觉问题，构建真正可用的知识问答系统。

| 知识点 | 与前几讲的关联 | 融合方式 |
| --- | --- | --- |
| Ollama 部署 | 第4讲的 OpenAI 兼容 API 调用 | 只改 base_url，代码完全复用 |
| 量化 | 第5讲的模型参数量概念 | 理解参数量→显存的换算 |
| RAG 三段式 | 第4讲的 Prompt Engineering | RAG 的 Prompt 模板 = PE 的进阶应用 |
| 向量化 | 第5讲的 BERT 编码器 | sentence-transformers 基于 BERT |
| Prompt 模板 | 第4讲的 System Prompt + 结构化输出 | 防幻觉 Prompt = 角色约束 + 引用格式 |

**Vibe Coding 融合点**：
- Spec-driven 部署规约（deploy_speckit.md）
- @文件引用让 AI 看到文档结构
- Inline Edit 快速改 chunk_size / top_k 对比效果

## 四、重点难点与应对策略

| 重点/难点 | 教学策略 |
| --- | --- |
| 重点：Ollama 完整操作流程 | 现场演示每一步，提供排查三步法 |
| 重点：递归字符分块策略 | 用真实文档演示不同 chunk_size 效果对比 |
| 重点：RAG 完整流水线串联 | 先分步讲每个环节，最后一个函数串联全部 |
| 难点：为什么要分块（三个原因） | 用"整篇 vs 分块"的检索效果对比直观展示 |
| 难点：向量化与余弦相似度 | 用"法国≈法兰西"的示例直观感受语义相似 |
| 难点：防幻觉 Prompt 设计 | 对比三种 Prompt 写法的输出差异 |

## 五、课前准备与课后作业

### 课前准备

1. **环境确认**：CloudStudio 中已安装 `openai`、`sentence-transformers`、`chromadb`、`markitdown`

3. **思考题**（二选一，写在实验报告末尾）
