## 一、实验信息

| 项目 | 内容 |

## 二、实验目的

1. 综合运用第1-7讲所学知识，独立设计并实现一个完整 Agent 系统
2. 掌握 LangChain 框架中 Agent 的构建方法（工具注册、ReAct 推理）
3. 理解 ReAct（Reasoning + Acting）范式的推理-行动循环机制
4. 学会将 RAG 检索能力封装为 Agent 可调用的工具（Tool）
5. 掌握 Agent 多轮对话记忆管理
6. 学会用 Gradio 构建可交互的智能问答界面

## 三、实验主题：你来定

本次实验没有固定的业务场景，**你来决定这个 Agent 要解决什么问题**。

它可以是：

- 一个本地运行的**学术论文助手**：帮你读 PDF、回答问题、整理要点
- 一个**代码审查 Agent**：接收代码文件，给出分析报告和改进建议
- 一个**个人知识库 Agent**：管理课程笔记、语义检索、自动摘要
- 一个**数据分析助手**：接收表格，回答数据问题并做简单计算
- 一个**旅行规划 Agent**：根据目的地文档回答交通、景点、预算问题
- 一个**课程答疑 Agent**：基于教材/讲义做智能问答
- 或者任何你觉得有价值的 AI 应用……

**唯一的约束**：你的 Agent 必须满足以下"技术骨架"要求（任务1-5），骨架之外的一切由你来填充。

> 如果实在想不到主题，可以使用提供的"航天产业报告.pdf"作为默认数据。

## 五、实验内容

### 任务1：RAG 工具封装（20分）

**目标**：将第6-7讲的 RAG 检索流水线封装为一个 LangChain Tool，使其能够被 Agent 调用。

**你需要做的**：
1. **选择你的知识库文档**（PDF / TXT / MD，至少一篇，内容与你的 Agent 主题相关）
2. 使用 markitdown 加载文档并转为文本
3. 使用递归字符分块（推荐 chunk_size=500, overlap=100）
4. 使用 sentence-transformers（推荐 all-MiniLM-L6-v2）进行向量化
5. 使用 ChromaDB 存储向量
6. 使用 `@tool` 装饰器封装检索函数
7. **工具描述必须清楚说明**：做什么、什么时候用、什么时候不用

**关键代码框架**：

```python

@tool

    """[在这里写清楚工具描述]
    - 触发条件：当用户问到关于[你的文档主题]的具体内容时
    - 功能：检索相关段落并返回
    - 边界：不适用于[不相关的场景]
    
    Args:
        query: 用户的检索问题
    """
    # 1. 对 query 进行向量化
    # 2. 在 ChromaDB 中检索 top-3 相关段落
    # 3. 返回拼接后的文本
    pass
```

**评分要点**：

| 评分项 | 分值 | 标准 |
|--------|:---:|------|
| RAG 流水线完整性（加载→分块→向量化→存储→检索） | 10分 | 每个环节运行无误 |
| 工具描述质量（触发条件+功能+边界） | 5分 | 描述清晰，Agent 能正确判断何时调用 |
| 工具可调用并返回正确结果 | 5分 | 测试检索返回相关内容 |

### 任务2：ReAct Agent 构建（25分）

**目标**：基于 LangChain 构建一个带 RAG 工具和至少一个辅助工具的 ReAct Agent。

**你需要做的**：
1. 使用 `ChatOpenAI` 连接 LLM（Ollama 本地或硅基流动 API）
2. 定义至少一个辅助工具（计算器 / 日期查询 / 文本摘要 / 自定义函数）
3. 编写 ReAct 格式的 Prompt 模板（包含角色定义、工具使用规范、Thought/Action/Observation 格式）
4. 使用 `create_react_agent` + `AgentExecutor` 创建 Agent
5. 设置 `verbose=True` 和 `max_iterations=5`

**工具选择建议**（根据你的主题选择）：

| Agent 主题 | 推荐辅助工具 |
|-----------|------------|
| 学术助手 | 引用格式化、关键词提取 |
| 数据分析 | 计算器、统计摘要 |
| 知识库 | 文本摘要、关键词搜索 |
| 通用 | 计算器（最简单的选择） |

**关键代码框架**：

```python
from langchain.agents import create_react_agent, AgentExecutor
from langchain_core.prompts import PromptTemplate

# 1. 初始化 LLM
llm = ChatOpenAI(model="qwen2.5:7b-instruct-q4_K_M", ...)

# 2. 定义计算器工具
@tool
def calculator(expression: str) -> str:
    """..."""
    pass

# 3. 工具列表
tools = [rag_search, calculator]

# 4. ReAct Prompt 模板
prompt = PromptTemplate.from_template("...")

# 5. 创建 Agent
agent = create_react_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools, ...)
```

**测试用例**（至少完成3个，围绕你自己的主题设计）：
1. 纯检索问题（只需调用 RAG 工具）
2. 需要辅助工具的问题（如计算、格式化）
3. 多步推理问题（需要多次工具调用才能回答）

**评分要点**：

| 评分项 | 分值 | 标准 |
|--------|:---:|------|
| Agent 构建完整性（LLM + 工具 + Prompt + Executor） | 10分 | 四要素齐全 |
| ReAct Prompt 格式正确性 | 10分 | 包含角色定义+格式约束+工具说明 |
| 3个测试用例通过 | 5分 | Agent 正确调用工具并给出合理回答 |

### 任务3：多轮对话记忆（25分）

**目标**：为 Agent 添加对话历史管理，使其能够理解上下文追问。

**你需要做的**：
1. 使用 `ChatMessageHistory` 管理消息历史
2. 使用 `RunnableWithMessageHistory` 包装 AgentExecutor
3. 实现 session 隔离（不同用户的对话互不影响）
4. 更新 Prompt 模板以包含 `{chat_history}` 占位符
5. 演示至少 3 轮连续对话（后续提问引用前面的内容）

**多轮对话设计思路**（根据你的主题设计）：

```
第1轮: 一个基础事实性问题（Agent 需要检索回答）
第2轮: 引用第1轮结果的追问（"这些里面哪个最重要？"）
第3轮: 基于第2轮的进一步追问（"帮我详细解释一下"）
```

关键是第2-3轮不重复完整信息，Agent 必须依靠记忆理解上下文。

**评分要点**：

| 评分项 | 分值 | 标准 |
|--------|:---:|------|
| ChatMessageHistory 实现正确 | 5分 | 消息正确存储和读取 |
| RunnableWithMessageHistory 包装正确 | 5分 | session 隔离生效 |
| Prompt 模板包含 chat_history | 5分 | 历史消息传入 Agent |
| 3轮连续对话演示（Agent 正确理解上下文） | 10分 | 第3轮能引用第1轮信息 |

### 任务5：Agent 行为分析（10分）

**目标**：记录并分析 Agent 的推理轨迹，理解 ReAct 范式的实际运行过程。

**你需要做的**：
1. 选择 1 个多步推理问题，记录完整的 Thought-Action-Observation 轨迹
2. 绘制该问题的 Agent 决策流程图（文本/Markdown/图片均可）
3. 分析：Agent 在哪些步骤做出了正确决策？是否有不必要的工具调用？
4. 思考：如果去掉某个工具，Agent 的行为会有什么变化？
5. 将第 1–4 项中的能力边界写成一个 `SKILL.md`：说明**何时触发**、**允许使用哪些工具**、**不得做什么**、**输出结构**和 **2–3 条验证提示词**。

**Skill 实践方法（必须参考随课 `skills/` 文件夹）**：
1. 先阅读 `../skills/README_zh.md`，了解模板、示例与规范入口。
2. 复制 `../skills/template/SKILL.md` 到你自己的实验目录：`skills/<skill-name>/SKILL.md`；不要直接修改随课模板。
3. 参考 `../skills/skills/skill-creator/SKILL.md` 的“明确意图 → 编写 → 测试 → 迭代”流程。若你的 Agent 要接 MCP 工具，可额外参考 `../skills/skills/mcp-builder/SKILL.md`。
4. 示例 Skill 的业务内容只作结构参考；你的 `name`、`description`、工作流、工具边界和验证提示词必须与自己的 Agent 主题一致。

**评分要点**：

| 评分项 | 分值 | 标准 |
|--------|:---:|------|
| 完整记录推理轨迹 | 4分 | 每一步 Thought/Action/Observation 完整 |
| 分析有深度 | 3分 | 不只是描述，而是评估决策质量 |
| `SKILL.md` 与 Agent 一致 | 3分 | 使用模板结构，触发条件、工具边界和验证提示词具体可检查 |

## 八、实验提示

1. **主题选择建议**：选你真正感兴趣的领域，报告会更有说服力。一个用心设计的"课程笔记助手"比勉强套用的"航天分析"得分更高。
2. **复用第6-7讲代码**：RAG 流水线的核心逻辑（分块、向量化、检索）直接复用，只需改文档和封装为 Tool。
3. **调试技巧**：设置 `verbose=True` 可以看到 Agent 的完整推理过程
4. **常见问题**：
   - Agent 不调用工具 → 检查工具描述是否清楚
   - Agent 无限循环 → 确认设置了 `max_iterations`
   - 对话历史不生效 → 确认 Prompt 中有 `{chat_history}` 占位符
5. **Skill 调试**：先用 2–3 条用户提示词检查 `description` 是否会在正确场景触发，再检查工作流、工具边界和输出是否符合预期；按 `skills/skills/skill-creator/SKILL.md` 的迭代思路记录一次改动。
6. **安全提示**：`calculator` 工具使用 `eval()` 存在安全风险，仅限教学演示使用
