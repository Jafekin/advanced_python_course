## 一、本讲教学目标

### 1.1 知识目标

**从 RAG 到 Agent 的演进**：
1. 理解 RAG 系统"检索+回答"的局限：无法执行动作、无法多步推理、无法自主决策
2. 理解 Agent 的核心定义：感知环境 → 自主决策 → 执行行动 → 观察反馈的闭环系统
3. 掌握 LLM Agent 的三大核心组件：规划（Planning）、工具（Tools）、记忆（Memory）

**工具调用（Tool Use）**：
4. 理解 Function Calling 协议：LLM 如何决定调用哪个工具、传什么参数
5. 掌握工具的三要素：名称（Name）、描述（Description）、执行逻辑（Function）
6. 掌握 LangChain 工具定义：`@tool` 装饰器、`BaseTool` 子类、工具绑定（`bind_tools`）
7. 理解工具描述对模型决策的关键影响：描述质量直接决定工具调用准确率

**多轮对话与状态管理**：
8. 理解消息历史的结构：system / human / ai / tool 四种消息角色
9. 掌握上下文窗口控制：滑动窗口截断、Token 计数、摘要压缩
10. 理解短期记忆（对话历史）与长期记忆（向量化存储）的区别与协作

**任务拆解与执行链**：
11. 掌握 ReAct 范式：Thought → Action → Observation 循环
12. 理解 Plan-and-Execute 范式：先规划完整计划，再逐步执行
13. 掌握 LangChain Agent 的构建：`create_react_agent` + `AgentExecutor`
14. 理解 Agent 的错误处理与最大迭代次数控制

**多 Agent 协作与通信协议**：
15. 理解多 Agent 协作的基本模式：角色分工、消息传递、任务分配
16. 理解 MCP（Model Context Protocol）：智能体与工具的标准化通信协议
17. 了解 A2A（Agent-to-Agent Protocol）：智能体间点对点协作协议
18. 了解 ANP（Agent Network Protocol）：大规模智能体网络的服务发现与路由

**Skill 与 Vibe Coding 工具总结**：
19. 理解 Skill 的概念：可复用的 Agent 能力单元（Prompt + 工具 + 约束）
20. 回顾八讲 Vibe Coding 工具体系：从 Prompt 迭代到 Agent Skills 的完整工具链

### 1.2 能力目标（Vibe Coding × Agent 开发）

1. 基于 LangChain 构建一个带工具调用能力的 ReAct Agent
2. 将第6-7讲的 RAG 检索能力封装为 Agent 工具，实现"能查资料的 Agent"
3. 为 Agent 添加多轮对话记忆，实现有状态的智能助手
4. 用 Gradio 构建可交互的智能问答助手界面（课程最终交付物）
5. 用 Spec-driven 方式管理 Agent 项目：写 agent_speckit.md 锁定 Agent 行为边界
6. 用 Agent Skills 思维审视整个课程 Vibe Coding 工具链的演进

### 1.3 态度目标

1. 建立"Agent 不是万能的"意识：理解 Agent 的能力边界（幻觉、工具调用失败、无限循环）
2. 形成"先定义工具边界再让 Agent 调用"的工程习惯
3. 理解"好的工具描述 = 好的 Agent 表现"，将 Prompt Engineering 延伸到工具层
4. 建立对 AI 系统安全性的基本意识：Agent 自主执行动作的风险与防护

## 四、Vibe Coding 工具讲授计划

### 4.1 本讲重点

① **Spec-driven Agent 行为规约（agent_speckit.md）**：
- 把 Agent 的行为边界写进 constitution：可用工具列表、禁止动作、输出格式
- 对比"自由 Agent" vs "Speckit 约束 Agent"的可控性差异

② **Prompt 工程在工具层的延伸**：
- 工具描述是另一种 Prompt——写给模型看的"工具说明书"
- 演示：同一工具不同描述 → Agent 调用行为的差异

③ **Agent Skills：以随课 `skills/` 目录为准的工程化写法**：
- Skill 是以 `SKILL.md` 为入口的可复用能力包：frontmatter 负责触发，Markdown 说明工作流、工具边界、输出与验证。
- 课堂依次阅读：`skills/README_zh.md`（导航）、`skills/template/SKILL.md`（最小模板）、`skills/skills/skill-creator/SKILL.md`（需求—测试—迭代）和 `skills/spec/agent-skills-spec.md`（规范入口）。
- 课堂操作：从模板复制到实验目录的 `skills/<skill-name>/SKILL.md`，写清楚触发条件、工具白名单、禁止动作、输出格式，并用 2–3 条测试提示词验证。
- 回顾课程中从 Prompt 迭代 → Rules → Spec → MCP → Agent Skills 的演进；只借鉴示例的结构与验证方法，不照搬无关工作流。

### 4.2 课程总结回顾

④ **八讲 Vibe Coding 工具链全景**：

| 讲次 | Vibe Coding 工具 | 核心能力 |
| --- | --- | --- |
| 第1讲 | Prompt 迭代、Rules、上下文工程 | 与 AI 高效对话的基础 |
| 第2讲 | Spec-driven（SpeckitList）、MCP | 用规约管理 AI 行为 |
| 第3讲 | System Prompt、上下文工程 | 控制模型输出的方向 |
| 第4讲 | @文件引用、Inline Edit | 让 AI 理解并修改代码 |
| 第6-7讲 | Spec-driven 部署、Inline Edit 调参 | 用 AI 管理工程项目 |
| 第8讲 | Agent Skills、MCP 协议 | AI 自主调用工具完成任务 |

## 五、重点难点与应对策略

| 重点/难点 | 教学策略 |
| --- | --- |
| RAG 和 Agent 的本质区别 | 类比：RAG 是"图书管理员"（只能查书回答），Agent 是"研究员"（能查书、计算、上网、写报告） |
| Function Calling 协议 | 拆解完整 JSON 流程：用户问题 → LLM 决定调用 → 返回函数名+参数 → 执行 → 结果回传 LLM |
| 工具描述影响 Agent 决策 | 现场演示：同一计算器工具，描述写"数学计算"vs"处理任何数字相关问题" → 调用频率差异 |
| ReAct 的循环何时终止 | 类比：人解题也是"想-做-看结果"循环，直到"我确定答案了"才停；max_iterations 防无限循环 |
| 多轮对话上下文窗口限制 | 数字直觉：4K tokens 约 3000 汉字，10 轮对话可能就爆了 → 需要滑动窗口或摘要 |
| MCP 和 A2A 的区别 | MCP 是"人和工具之间的 USB-C 接口"，A2A 是"人和人之间的对讲机" |
| Skill 的具体写法 | 类比：Skill 就像一张"岗位说明书"——角色定位 + 可用工具 + 行为规范 + 输出格式 |

   ```bash
   ollama serve &
   ollama pull qwen2.5:7b-instruct-q4_K_M
   ```
3. 回顾第6-7讲的 RAG 流水线实现（分块、向量化、检索、生成）
4. 回顾第3讲的 OpenAI 兼容 API 调用方式
5. 浏览 `第8讲/skills/README_zh.md`，并打开 `skills/template/SKILL.md` 与一个示例（建议 `skills/skills/skill-creator/SKILL.md`）；课堂中将以这些本地文件为准设计 Skill。

### 6.2 课后作业：实验八（基于航天产业报告的智能问答 Agent）

**实验数据**：`航天产业报告.pdf`（中国银河证券《2024 年将是中国商业航天的井喷之年》行业点评，复用第6-7讲数据）

| 任务 | 内容 | 知识点 | 分值 |
| --- | --- | --- | --- |
| 任务1（20分） | RAG 工具封装 | 将第6-7讲 RAG 流水线封装为 LangChain Tool | 20 |
| 任务2（25分） | ReAct Agent 构建 | 基于 LangChain 构建带 RAG 工具的 ReAct Agent | 25 |
| 任务3（25分） | 多轮对话记忆 | 为 Agent 添加对话历史管理，支持上下文追问 | 25 |
| 任务4（20分） | Gradio 交互界面 | 构建可交互的智能问答助手，支持多轮对话 | 20 |
| 任务5（10分） | Agent 行为分析 + Skill 设计 | 分析 Agent 轨迹，并以提供的 `skills/` 模板写出能力边界 | 10 |

**实验要求**：
1. 基于 LangChain 框架实现（本讲重点框架）
2. RAG 工具必须复用第6-7讲的向量化+检索逻辑
3. 实验报告需包含至少 3 轮完整的 Agent 推理轨迹截图
4. Gradio 界面需支持清除对话历史的功能
5. 提交一个自定义 `SKILL.md`：以 `第8讲/skills/template/SKILL.md` 为起点，存放在实验目录的 `skills/<skill-name>/SKILL.md`；报告说明参考过的本地 Skill 文件和 2–3 条验证提示词。

### 6.3 拓展（选做）

- 为 Agent 添加更多工具（如计算器、日期查询），观察多工具场景下的决策行为
- 实现简化版 Plan-and-Execute Agent，对比与 ReAct 的行为差异
- 尝试用 MCP 协议封装一个简单的工具服务器
