## 一、本讲教学目标

### 1.1 知识目标

1. 理解自然语言处理（NLP）的基本任务与大模型在其中的角色
2. 掌握分词（Tokenization）原理：为什么需要分词、子词分词思想、BPE 算法的合并过程
3. 理解 Token 对开发者的实际意义：上下文窗口、计费、Prompt 长度管理
4. 掌握大模型采样参数：Temperature、Top-k、Top-p 的原理与调参策略
5. 掌握 Prompt Engineering 核心技巧：Zero/One/Few-shot、角色扮演、思维链（CoT）、结构化输出
6. 能通过 OpenAI 兼容 API 调用大模型，完成文本生成、分类、信息抽取等任务
7. 了解模型选择的关键考量因素（性能/成本/速度/上下文长度）
8. 理解缩放法则（Scaling Laws）与模型幻觉的成因及缓解策略

### 1.2 能力目标（Prompt Engineering 实战）

1. 能根据任务类型选择合适的 Prompt 策略（Zero-shot vs Few-shot vs CoT）
2. 能设计 System Prompt 控制模型的角色、风格和输出格式
3. 能使用 tiktoken 计算 Token 数，合理管理上下文预算
4. 能识别模型幻觉并采用合适的缓解手段
5. 能将 Vibe Coding 中的 Prompt 技巧迁移到通用 LLM 应用开发

### 1.3 态度目标

1. 建立"Prompt 即编程"的认知：Prompt 设计是一种新型编程范式
2. 理解大模型的能力边界，既不神化也不轻视
3. 形成"先理解原理，再工程化应用"的学习习惯

## 三、知识线与工具线融合设计

**【核心设计思路】** 本讲是"从使用者到理解者"的转折点：前两讲学生已经在用 Prompt 驱动 AI 写代码，本讲让他们系统理解 Prompt 背后的原理，并将技巧从 Vibe Coding 拓展到通用 LLM 应用开发。

|  |  |  |
| --- | --- | --- |
| **知识点** | **与前两讲的关联** | **融合方式** |
| 分词与 Token | 第2讲 Speckit 中提到的 Token 限制 | 实操 tiktoken 计算 Token 数，理解上下文窗口限制 |
| 采样参数 | 第1讲 AI 生成代码的"随机性"体验 | 理解为什么同样的 Prompt 有时输出不同 |
| Prompt Engineering | 第1讲两段式 Prompt 的理论升级 | 从"先氛围后约束"扩展到完整的 Prompt 设计方法论 |
| API 调用 | 第1讲 MCP 概念的具体落地 | 用代码直接调用大模型，为第4讲 RAG 做铺垫 |
| 模型选择 | 第2讲选用不同 sklearn 模型的类比 | 不同模型有不同的能力-成本权衡 |
| 模型幻觉 | 第1讲"Diff 审查"习惯的深层原因 | 理解为什么必须审查 AI 输出，引出第4讲 RAG |

## 四、重点难点与应对策略

|  |  |
| --- | --- |
| **重点/难点** | **教学策略** |
| 难点：BPE 算法的合并过程 | 用 "hug/pug/pun/bun" 迷你语料手动演示合并步骤，再用代码验证 |
| 重点：Temperature/Top-p 参数调优 | 同一 Prompt 用不同参数对比输出，让学生亲眼看到差异 |
| 重点：Few-shot vs CoT 的选择 | 给出决策树：简单分类→Few-shot；需要推理→CoT；格式严格→结构化输出 |
| 难点：System Prompt 设计 | 提供三个模板（分类器/翻译器/分析师），学生修改后对比效果 |
| 难点：模型幻觉的识别 | 展示真实幻觉案例（虚构论文引用、错误代码逻辑），培养批判性思维 |
| 重点：API 调用的工程化 | 从 curl 到 Python SDK，强调 messages 结构和流式输出 |

### 5.2 课后作业：实验三

实验任务说明见 `实验三/` 目录：

| 任务 | 内容 | 核心技巧 | 分值 |
|------|------|---------|------|

**思考题（三选一）**：

a. Temperature=0 和 Temperature=1.5 分别适合什么场景？如果你要做一个代码生成助手，应该怎么设置采样参数组合？

b. 思维链（CoT）为什么能提升推理准确率？它有什么局限性？在什么场景下反而会降低性能？

c. 大模型的"幻觉"问题在实际应用中如何检测？你认为 RAG 和 CoT 自我验证哪种方式更适合"事实性问答"场景？
