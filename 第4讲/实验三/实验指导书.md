## 四、实验任务

### 任务一：BPE 分词算法实现（40分）

**背景**：BPE（Byte-Pair Encoding）是 GPT、DeepSeek、Qwen 等大模型采用的分词算法核心，掌握它有助于理解模型如何"看到"文字。

**要求**：

**Step 1**：实现 `train_bpe(corpus, num_merges)` 函数

- 输入：`corpus`（`dict`，词→频率）、`num_merges`（合并次数）
- 功能：执行 BPE 训练，每步打印合并的 token 对
- 输出：`(merges, vocab)` — 合并历史列表 + 最终分词结果

**Step 2**：在以下语料上训练（目标词表大小 12）：

```python
corpus = {"study": 3, "student": 5, "studio": 2, "state": 4, "stable": 2, "stem": 3}
```

**Step 3**：实现 `bpe_encode(word, merges)` 函数，对未见过的词进行分词

**Step 4**：对以下单词分词并打印结果：

```python
test_words = ["students", "studies", "statement", "unstable"]
```

**Step 5**：用 `tiktoken` 对相同单词分词，对比结果异同并分析原因

**评分标准**：

- `train_bpe` 实现正确，合并步骤输出完整（20分）
- `bpe_encode` 实现正确（8分）
- tiktoken 对比与原因分析（12分）

### 任务二：Prompt Engineering 对比实验（60分）

**背景**：不同的 Prompt 策略对模型输出的质量和稳定性影响显著，本任务通过量化对比建立直觉。

**数据**：使用 `reviews.json`，取前 10 条（id 1-10）作为测试集，`label` 字段为 ground truth。

**要求**：

**Step 1**：设计三组 Prompt，对同一批数据完成情感分类（正面/负面/中性）：

| 组别  | 策略        | 要求                               |
| --- | --------- | -------------------------------- |
| A组  | Zero-shot | 只给任务描述，不给示例                      |
| B组  | Few-shot  | 给 3 个示例（正/负/中性各1条），示例**不得**来自测试集 |
| C组  | CoT       | 要求模型先列出情感词，再判断，最后输出结论            |

**Step 2**：统计并填写对比表格：

| 指标              | A组（Zero-shot） | B组（Few-shot） | C组（CoT） |
| --------------- | ------------- | ------------ | ------- |
| 准确率（与 label 对比） |               |              |         |
| JSON 解析成功率      |               |              |         |
| 平均 Token 消耗     |               |              |         |
| 平均响应时间（秒）       |               |              |         |

> 提示：要求三组 Prompt 的输出均为 JSON 格式 `{"sentiment": "正面/负面/中性"}`；用 `tiktoken` 统计 Token；用 `time.time()` 统计响应时间。

**Step 3**：写出不少于 100 字的结论分析：哪种策略在本任务上最优？为什么？

**评分标准**：

- 三组实验代码实现正确（25分）
- 对比表格数据完整（20分）
- 结论分析有依据、逻辑清晰（15分）

## 六、常见问题

**Q：API 调用返回 402 错误怎么办？**
A：API Key 额度耗尽，联系老师补充额度；或减少调用次数（任务二先用 5 条数据测试）。

**Q：JSON 解析失败怎么处理？**
A：在 Prompt 中加强约束（"不要添加任何其他内容"）；或用正则表达式提取 JSON 部分；设置 `temperature=0` 提高稳定性。

**Q：BPE 实现的 `token` 对用什么数据结构存？**
A：用 `tuple`，例如 `('s', 't')` 代表合并 `s` 和 `t`；`merges` 是 `list of tuple`，保存合并顺序。

**Q：任务二的 Few-shot 示例可以用 reviews.json 里的数据吗？**
A：不可以。Few-shot 示例必须是独立编写的，不能来自测试集，否则会导致数据泄露。
