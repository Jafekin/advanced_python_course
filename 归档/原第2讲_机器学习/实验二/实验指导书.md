## 三、Spec-driven 核心概念（实验前必读）

### 什么是 Speckit？

Speckit 是 GitHub 开源的 Spec-driven 开发工具，核心理念是**先写规约，再让 AI 按规约执行**。

### speckit.md 的三段式结构

```markdown
# 项目名称

## constitution（不可违反的硬约束）
- 约束1：...
- 约束2：...

## context（数据背景 + 分析目标）
数据：...
字段：...
目标：...

## 任务列表（可执行、可验证的步骤）
- [ ] 任务1：...
- [ ] 任务2：...
```

### 使用方式

每次执行任务时，在 Prompt 中附带 `@speckit.md` 引用：
```
@speckit.md 执行任务1：加载数据并打印形状
```

AI 会自动遵守 constitution 中的所有约束，无需每次重复。

## 四、实验1：线性回归 + 简单规约（25分）

### 4.1 理论回顾

**线性回归**：y = ax + b，通过最小化 MSE 找到最佳参数

| 指标 | 说明 |
|---|---|
| RMSE | 均方根误差，与 y 同量纲，越小越好 |
| R² | 判定系数，越接近 1 越好 |

**reshape(-1, 1)**：sklearn 要求 X 是二维矩阵，单特征需转为列向量

### 4.2 数据说明

`load_diabetes()` 包含 442 名糖尿病患者的 10 项生理指标（已标准化），目标值为一年后的疾病进展量化值。本实验选取 BMI（体质指数）单特征进行简单线性回归。

```python
from sklearn.datasets import load_diabetes
diabetes = load_diabetes()
# diabetes.data[:, 2] 是 BMI 特征
# diabetes.target 是疾病进展值
```

## 五、实验2：SVM 分类 + 规约驱动（30分）

### 5.1 理论回顾

**SVM（支持向量机）**：找到最大间隔的超平面进行分类

| 概念 | 说明 |
|---|---|
| 核函数 | 将数据映射到高维空间，RBF 核最常用 |
| F1-score | 精确率与召回率的调和平均，多分类用 micro/macro |
| 决策边界 | 分类器在特征空间中的分界面 |

**准确率陷阱**：高准确率不代表模型好，必须看 F1-score

### 5.2 数据说明

`load_wine()` 包含 178 个红酒样本的 13 项化学成分（酒精度、苹果酸、灰分、镁含量等），目标是分类为 3 种品种（cultivar）。

```python
from sklearn.datasets import load_wine
wine = load_wine()
# wine.data: (178, 13)
# wine.target: 0/1/2 三种品种
# wine.feature_names: 13个化学特征名
```

### 5.3 实验任务

**Step 1**：编写 `speckit_exp2.md`，包含：
- constitution：至少 3 条 ML 约束
- context：描述红酒数据集和分类目标
- 任务列表：3 条可执行任务

**Step 2**：用 `@speckit_exp2.md` 引用，驱动 CodeBuddy 完成：

2. 训练 SVM 分类器（rbf 核），输出 F1-score

### 5.4 决策边界辅助函数

以下两个函数可直接使用，也可让 AI 生成：

```python
def make_meshgrid(x, y, h=.02):
    x_min, x_max = x.min() - 1, x.max() + 1
    y_min, y_max = y.min() - 1, y.max() + 1
    xx, yy = np.meshgrid(np.arange(x_min, x_max, h),
                         np.arange(y_min, y_max, h))
    return xx, yy

def plot_contours(ax, clf, xx, yy, **params):
    Z = clf.predict(np.c_[xx.ravel(), yy.ravel()])
    Z = Z.reshape(xx.shape)
    out = ax.contourf(xx, yy, Z, **params)
    return out
```

## 六、实验3：乳腺癌数据聚类与降维验证（45分）

### 6.1 理论回顾

**K-means 聚类**：将数据划分为 K 个簇，无需标签

| 概念 | 说明 |
|---|---|
| 肘部法则 | 绘制 inertia 随 K 变化曲线，找拐点确定最佳 K |
| K-means++ | 改进的初始化方法，避免随机初始化的不稳定 |
| inertia_ | 各样本到聚类中心的距离平方和（越小越好） |

**PCA 降维**：将高维数据投影到低维空间，保留最大方差方向

| 概念 | 说明 |
|---|---|
| StandardScaler | PCA 前必须标准化，否则大值特征主导主成分 |
| explained_variance_ratio_ | 各主成分解释的方差比例 |
| 累计解释方差 | 前 k 个主成分保留的信息量占比 |

### 6.2 数据说明

`load_breast_cancer()` 包含 569 例乳腺肿瘤样本的 30 项细胞核特征（半径、纹理、周长、面积、光滑度等的均值/标准差/最大值），真实诊断标签为恶性（0）/良性（1）。

本实验**忽略诊断标签**，仅根据 30 项特征进行无监督聚类，再用 PCA 降维可视化验证聚类质量。

```python
from sklearn.datasets import load_breast_cancer
cancer = load_breast_cancer()
# cancer.data: (569, 30)
# cancer.target: 0(恶性)/1(良性) — 本实验不用于训练，仅用于最终验证
# cancer.feature_names: 30个特征名
```

## 十、常见问题

**Q：speckit.md 的 constitution 写什么？**
A：写可检查的硬约束，而非口号。好的约束："必须设置 random_state=42"；坏的约束："代码要规范"。

**Q：reshape(-1, 1) 报错？**
A：检查数据是否已经是 numpy 数组，用 `np.array(x).reshape(-1, 1)`。

**Q：肘部曲线看不出拐点？**
A：找 loss 下降速度突然变缓的位置。乳腺癌数据 K=2 处通常拐点明显（对应恶性/良性两类）。

**Q：PCA 不先标准化？**
A：乳腺癌数据 30 个特征量纲差异极大（面积上千 vs 光滑度小于1），不标准化会导致面积类特征完全主导主成分。

**Q：决策边界函数不会写？**
A：把 make_meshgrid 和 plot_contours 两个函数发给 AI，让它解释并应用到你的模型。

**Q：@speckit.md 引用后 AI 没遵守约束？**
A：检查文件名是否正确、constitution 部分格式是否规范（每条约束独占一行，用 `-` 开头）。

**Q：红酒数据 13 个特征怎么选两个画决策边界？**
A：选区分度高的两个特征（如酒精度 alcohol + 颜色强度 color_intensity），或用 PCA 降到 2 维后画。
