## 一、本讲教学目标

### 1.1 知识目标

**理论基础**：
1. 理解人工智能、机器学习、深度学习的关系
2. 掌握机器学习的核心工作流程：数据→特征工程→模型训练→评估→部署
3. 理解监督学习 vs 无监督学习的本质区别
4. 掌握特征工程的概念：特征提取、转换、选择、降维

**算法理论**：
5. 掌握线性回归：损失函数（MSE）、梯度下降法、正规方程解
6. 掌握 SVM 分类：最大间隔、核函数（线性/多项式/RBF）、Hard/Soft Margin
7. 掌握 K-means 聚类：算法原理、迭代过程、肘部法则
8. 掌握 PCA 降维：主成分分析、协方差矩阵、特征值分解、解释方差比

**评估指标**：
9. 掌握回归评估：MSE、RMSE、MAE、R²
10. 掌握分类评估：混淆矩阵、精确率、召回率、F1-score、准确率陷阱

### 1.2 能力目标（Vibe Coding × 机器学习）

1. 用自然语言描述建模需求，由 AI 生成完整训练流程代码（两段式 Prompt 建模节奏）
2. 用 @文件引用 + Prompt 完成数据加载与预处理，不手写 sklearn 样板代码
3. 用 Inline Edit 快速切换算法或评估指标，建立 Diff 审查习惯
4. 用 AI 完成决策边界、肘部法则曲线等可视化的自动生成
5. 编写 speckit.md 规约文件，用 Spec-driven 方式管理多步骤 ML 项目
6. 形成"AI 生成 → 自己验证"的协作建模工作流

### 1.3 态度目标

1. 建立"工程化建模"思维：不要求推导公式，要求能端到端跑通并解释结果
2. 形成"生成→验证→迭代"的 AI 协作建模习惯，而非盲目接受 AI 输出
3. 理解"监督学习 vs 无监督学习"的本质区别，能根据任务选择算法类型
4. 认识到规约驱动（Spec-driven）在复杂项目中的价值：先定义约束，再让 AI 执行

## 四、Vibe Coding 工具建模讲授计划

### 4.1 本讲深度讲解（重点）

① **两段式 Prompt（建模节奏）**：Round 1 描述任务类型和数据，Round 2 指定算法、评估指标、输出格式

② **Rules 中加入 ML 规范**：
```markdown
# ML 建模规范（追加到 python-style.md）
- 数据建模前必须对特征做 StandardScaler 归一化（PCA、SVM、KNN 等）
- 划分数据集必须设置 random_state，保证结果可复现
- 评估分类模型必须输出 F1-score，禁止只看 accuracy
- 聚类任务必须先用肘部法则确定 K 值，不能凭感觉拍
```

③ **@文件引用（ML 场景）**：引用 data.xlsx / mall_customers.csv 让 AI 了解列名

④ **Inline Edit（Cmd+K）**：在已有代码上局部修改（换算法、加可视化、改评估指标）

⑤ **Diff 审查（ML 专项）**：检查 fit/predict 数据是否混用训练集测试集、reshape 维度是否正确

### 4.2 本讲深度讲解（进阶）

⑥ **Speckit for ML（规约驱动建模）**：
- 什么是 Spec-driven 开发：先写规约、再让 AI 按规约执行
- speckit.md 结构：constitution（底线约束）+ context（数据背景）+ 任务列表
- ML 项目中的 constitution：防数据泄露、强制 random_state、指定评估指标
- @speckit.md 引用：每次对话自动携带约束，AI 全程遵守

⑦ **Speckit vs 两段式 Prompt 选型**：单任务用两段式 Prompt，多步骤项目用 Speckit 管全局约束

## 五、重点难点与应对策略

|  |  |
| --- | --- |
| **重点/难点** | **教学策略** |
| 难点：理解 fit 只能在训练集上调用 | 演示数据泄露的错误示例，对比正确 Pipeline 写法 |
| 难点：SVM 决策边界的网格绘制 | 提供现成的 make_meshgrid + plot_contours 函数，用 Prompt 让 AI 解释 |
| 重点：肘部法则如何选 K | 画 loss 随 K 变化曲线，找"拐点最明显"处 |
| 难点：PCA 为什么要先标准化 | 演示不标准化的后果（大值特征主导主成分方向） |
| 重点：分类 vs 聚类的区别 | 类比：分类有标准答案（监督），聚类没有答案（无监督）|
| 难点：reshape(-1, 1) 的含义 | 解释：sklearn 要求 X 是二维矩阵，单特征也要转成列向量 |
| 难点：梯度下降法原理 | 用"下山"比喻：学习率=步长，梯度=最陡方向 |
| 难点：核函数的作用 | 解释：将数据映射到高维空间，使线性不可分变为线性可分 |
| 重点：混淆矩阵与 P/R/F1 | 用癌症预测场景举例（99.9% 准确率的陷阱）|
| 重点：speckit.md 的 constitution 怎么写 | 给出 ML 场景的标准模板，强调"可检查的约束"而非口号 |
| 难点：Speckit 与两段式 Prompt 的选型 | 对比表 + 实验3 的实际体验让学生直观感知差异 |

**实验2：SVM 分类 + 规约驱动（红酒数据集）**

**实验3：乳腺癌数据聚类与降维验证（乳腺癌数据集）**
