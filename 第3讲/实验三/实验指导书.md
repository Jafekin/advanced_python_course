```

- visualizer.py: 可视化（条形图/折线图/科研图表）
加上 __init__.py 导出公开接口。
保持所有功能不变，每个函数有 docstring。
```

## 任务2：类型注解 + 异常处理（25分）

### 目标

为核心模块添加类型安全和错误处理能力。

### 具体要求

**2.1 类型注解（12分）**

为 `loader.py` 和 `cleaner.py` 的所有公开函数添加完整类型注解：

```python
from pathlib import Path
from typing import Optional, Any

def load_data(
    file_path: Path,
    sheet_name: Optional[str] = None,
    encoding: str = "utf-8"
) -> pd.DataFrame:
    ...
```

- 参数和返回值均标注
- 使用 `Optional` 标记可选参数
- 复杂返回用 `tuple[...]` 标注

**2.2 自定义异常（6分）**

创建 `sci_analyzer/exceptions.py`：

```python
class DataError(Exception): ...
class LoadError(DataError): ...
class CleanError(DataError): ...
class StatsError(DataError): ...
```

**2.3 异常处理（7分）**

为 `loader.py` 添加完整错误处理链：
- 文件不存在 → `FileNotFoundError`
- 格式不支持 → `ValueError`
- 编码错误 → 自动降级为 gbk 重试
- 其他错误 → 包装为 `LoadError`

### Vibe Coding 操作

```
@sci_analyzer/loader.py
@sci_analyzer/exceptions.py
请为 loader.py 的所有函数添加：
1. 完整类型注解（Path, Optional, pd.DataFrame）
2. 参数校验（文件存在性、格式检查）
3. try/except 链：FileNotFoundError → UnicodeDecodeError(降级gbk) → LoadError
4. 不使用 bare except
```

### 评分标准

| 评分项 | 分值 | 标准 |
|--------|:---:|------|
| 类型注解完整 | 6分 | 所有公开函数参数+返回值 |
| 类型使用正确 | 6分 | Optional/Union/tuple 使用合理 |
| 异常类层级合理 | 6分 | 基类+3个子类，继承正确 |
| 错误处理覆盖全 | 7分 | 文件/编码/格式/通用错误均处理 |

### 目标

### 具体要求

- 测试 Excel 加载成功
- 测试文件不存在的错误处理
- 测试不支持格式的错误处理

**3.2 Fixture 使用（8分）**

```python
@pytest.fixture
def sample_dataframe():
    """通用测试数据"""
    return pd.DataFrame({
        'name': ['A', 'B', 'C', 'A'],
        'value': [1.0, 2.0, np.nan, 1.0],
        'category': ['x', 'y', 'x', 'x']
    })
```

- 用 fixture 提供测试数据
- 用 `tmp_path` fixture 创建临时测试文件

**3.3 参数化测试（7分）**

```python
@pytest.mark.parametrize("method,expected", [

    ...
```

- 至少一组参数化测试
- 覆盖 cleaner 模块的不同策略

## 任务4：Spec-driven 新功能（30分）

### 目标

### 具体要求

```markdown
# speckit.md

## Constitution 引用
（列出 5+ 条硬约束规则）

（项目环境、已有模块、数据来源）

### 功能要求
```
