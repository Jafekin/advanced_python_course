## 一、本讲教学目标

### 1.1 知识目标

1. 理解基础 RAG 的三大痛点（用词不匹配/语义鸿沟/排序不精）
2. 掌握四种高级检索策略：MQE 多查询扩展、HyDE 假设文档嵌入、Reranker 重排序、混合检索
3. 掌握 RAG 评估指标：Recall@k、Precision@k、MRR 的定义和计算
4. 理解 chunk_size / top_k / 嵌入模型对检索质量的影响
5. 理解 Agent 四类型记忆体系（工作/情景/语义/感知）
6. 掌握基于 JSON + 向量检索的简化记忆系统实现
7. 理解 Memory 与 RAG 的关系和融合方式

### 1.2 能力目标

1. 能实现 MQE 多查询扩展并对比检索效果
2. 能实现 Reranker 两阶段检索并量化精度提升
3. 能构建评估数据集并计算 Recall@k / MRR
4. 能实现简单的 Agent 记忆系统（语义记忆 + 情景记忆）
5. 能将记忆系统与 RAG 融合，构建"既懂你又懂事"的问答 Agent

### 1.3 态度目标

1. 建立"评估驱动优化"的工程意识——不靠感觉，靠数据
2. 认识到 Agent 记忆是人机交互从"工具"到"伙伴"的关键
3. 形成"先评估、再优化、后验证"的迭代思维

## 三、知识线与工具线融合设计

**【核心设计思路】** 本讲是"从能跑到跑得好"的深化：第6讲搭了基础 RAG，本讲用评估数据说明问题，用高级策略系统性解决，并为第8讲 Agent 系统铺设记忆基础。

| 知识点 | 与前几讲的关联 | 融合方式 |
| --- | --- | --- |
| MQE 多查询扩展 | 第4讲 Prompt Engineering 改写 | 用 LLM 生成改写查询 = PE 的一种应用 |
| HyDE 假设文档 | 第4讲 CoT / 第6讲向量化 | LLM 生成假设答案 + 向量化检索 |
| Reranker | 第5讲 BERT Cross-Encoder | 复用 Transformer 编码器做精排 |
| 评估指标 | 第3讲 pytest 测试思维 | "先写测试再实现" → "先定指标再优化" |
| Agent 记忆 | 第6讲 ChromaDB / Prompt 模板 | 记忆检索复用 RAG 技术栈 |

**Vibe Coding 融合点**：
- Prompt 调优：设计 MQE 的改写 Prompt
- Inline Edit：快速改 chunk_size / top_k 对比效果
- 上下文工程：Memory context 注入 System Prompt

## 四、重点难点与应对策略

| 重点/难点 | 教学策略 |
| --- | --- |
| 重点：MQE 和 Reranker 的代码实现 | 每策略先讲原理（1张），再给完整代码（1张），再跑效果对比（1张） |
| 重点：评估数据集构建 | 现场演示从已有 chunk 反向生成问题的方法 |
| 重点：记忆系统的存储和检索 | 用真实对话场景演示"记住/回忆"全流程 |
| 难点：RRF 融合公式 | 用排名表格手算一遍，再看代码 |
| 难点：记忆写入时机（什么值得记） | 给出明确规则：姓名/偏好/目标必记，闲聊不记 |
| 难点：Memory 与 RAG 如何在 Prompt 中合并 | 给出模板：System Prompt = 角色 + 记忆 + RAG 知识 |
