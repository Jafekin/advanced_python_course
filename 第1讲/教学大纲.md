## 一、本讲教学目标

### 1.1 知识目标

1. 掌握 Pandas Series 的创建（列表/字典/常量）、索引访问（标签/位置/布尔）、统计方法全景（sum/mean/std/median/describe/value_counts/quantile）及时间序列操作（shift/rolling）
2. 掌握 Pandas DataFrame 的创建、选列、筛选、排序、新增列，以及分组聚合（groupby）、宽转长（melt）、Top N（nlargest）
3. 掌握 CSV/Excel（多 sheet）/JSON 文件的读写，缺失值的两种策略（dropna vs fillna）及编码处理（utf-8-sig）
4. 掌握 Matplotlib 图表决策树（分布→直方图、对比→柱状图、关系→散点图、趋势→折线图、分组→箱线图）、多子图布局、3D散点图及科技感风格迭代
5. 掌握 NetworkX 社交网络图的构建、布局、最短路径分析

### 1.2 能力目标（Vibe Coding 工具链）

1. 掌握两段式 Prompt 节奏：先描述感觉，再逐步加约束

2. 配置并使用 Rules 规则文件，锁定 AI 输出风格

3. 理解上下文四层级，用 @文件引用 精准注入数据结构

4. 使用 Inline Edit（Cmd/Ctrl+K）进行行内修改，建立 Diff 审查习惯

5. 了解 MCP 协议的作用和配置方式

6. 了解 Agent Skills 和自定义斜杠指令的封装逻辑

### 1.3 态度目标

1. 建立"意图优先于语法"的 Vibe Coding 思维方式

2. 形成"先看 Diff 再 Accept"的代码审查习惯

3. 对 AI 生成的代码保持判断力，既不盲目接受，也不因不信任而放弃使用

## 三、两条主线融合设计

**【核心设计思路】** 不把"知识讲完再讲工具"，而是每个 Pandas 知识点就是对应工具的使用场景

|  |  |  |
| --- | --- | --- |
| **Pandas 知识点** | **Vibe Coding 工具** | **融合方式** |
| Series 创建与统计（含时间序列 shift/rolling） | 两段式 Prompt | 学生通过两轮 Prompt 完成任务，对比一次性写完的差距 |
| DataFrame 筛选/排序/分组聚合/melt | Rules 规则文件 | 先配 Rules，再生成代码，验证规范是否生效 |
| Excel 多 sheet 读取与缺失值策略 | @文件引用 + 上下文四层级 | 演示有/无 @引用时 AI 生成代码的差异 |
| 新增计算列（向量化） | Inline Edit (Cmd+K) | 用行内编辑修改筛选阈值，不切换到 Chat 窗口 |
| 数据可视化（图表决策树+四类图表） | Diff 审查习惯 | 三轮迭代，每轮 Accept 前完成 Diff 三步清单 |

## 四、Vibe Coding 工具全景讲授计划

### 4.1 本讲深度讲解（重点）

① 两段式 Prompt：先氛围后约束——贯穿全讲每个知识点

② Rules 规则文件：配置路径、规则写法、验证方法、规则优先级

③ 上下文四层级：系统层/会话层/指令层/隐式层的概念与控制方式

④ @文件引用：操作步骤、必须手动输入@、最小上下文原则

⑤ Inline Edit：Cmd+K 三种模式、Diff 审查三步清单

### 4.2 本讲概览介绍（预告）

⑥ MCP：模型上下文协议，AI 感官延伸，连接外部数据源

⑦ Agent Skills：把临时 Prompt 升级为可复用 SOP

⑧ 自定义斜杠指令：高频工作流封装为一行命令

## 五、重点难点与应对策略

|  |  |
| --- | --- |
| **重点/难点** | **教学策略** |
| 难点：两段式 Prompt 的"感觉"第一轮很难写 | 提供3个模板例句： 1. "创建一个...类型的..." 2. "帮我用...方式..." 3. "我想要一个...的效果" |
| 难点：理解上下文四层级的控制逻辑 | 现场演示：同样问题，有/无 @引用时 AI 给出不同代码的对比 |
| 重点：建立 Diff 审查习惯 | 每次演示改代码后，带学生读 Diff，念出三步清单 |
| 难点：区分 Rules/Skills/MCP 的职责 | 类比：Rules=衣服风格, Skills=执行SOP, MCP=连接互联网 |
| 重点：向量化操作的正确使用 | 同一功能的 for 循环 vs 向量化版本并排对比 |

**任务1：航天贸易数据分析（航天进出口额.xlsx）**

数据集：国家统计局海关总署「HS88章 航空器、航天器及零件」月度进出口额（76 行 × 29 列，含 42 个国家/地区，2022-01 ~ 2023-12 共 24 个月，单位万元）

指标列需解析：`进口额(人民币)_(HS88章)航空器、航天器及其零件_日本_当期`，用 `split('_')` 拆出方向、国家。

要求：用两段式 Prompt + @文件引用，完成贸易分析与三类图表

Round 1（氛围层）：
```
读取 航天进出口额.xlsx，展示数据基本信息，
解析指标列拆出方向和国家，清洗缺失值，
用柱状图、折线图和箱线图展示贸易分布
```

Round 2（约束层）：
```
在保持上面代码的前提下，按要求调整：
1. 解析指标列：用 split('_') 拆出方向、国家
2. 筛选进口额，计算各国 24 个月合计，取 Top10
3. 柱状图：Top10 进口来源国，加数值标签
4. 折线图：某国（如日本）24 个月趋势，加标题和轴标签
5. 箱线图：Top5 国家月度分布对比，红色标注异常值
6. 全部图表支持中文显示
```

**任务2：社交网络可视化（karate.gml）**

数据集：Zachary 空手道俱乐部社交网络（34成员 x 78关系）

Round 1（氛围层）：
```
读取 karate.gml，用 NetworkX 绘制社交网络图，节点显示标签，spring_layout 布局
```

Round 2（约束层）：
```
在保持上面代码的前提下：
1. 计算并打印节点数、边数、平均度数
2. 柱状图展示每个节点到其他节点的平均最短路径长度
3. 直方图展示10号节点到其他可达节点的距离分布，标注平均值
4. 红色高亮显示节点1到节点10的最短路径，其余边灰色
5. 全部图表支持中文显示
```

Diff 审查要点：节点标签是否为整数（gml 默认字符串，需 relabel）；最短路径算法方向是否正确

**任务3：全球机场 3D 可视化（airports.json）**

数据集：全球约5万个机场，嵌套JSON结构
```json
{ "00AK": { "lat": 59.949, "lon": -151.696, "elevation": 450 } }
```
注意：elevation 单位是英尺，需转米（x0.3048）

Round 1（氛围层）：
```
读取 airports.json，提取所有机场的 lat/lon/elevation，
用 Matplotlib 3D 散点图展示全球机场空间分布
```

Round 2（约束层）：
```
在保持上面代码的前提下：
1. 海拔从英尺转为米（x0.3048）
2. 轴范围：经度-200~200，纬度-60~60，海拔0~4000
3. 添加轴标签（经度/纬度/海拔（米））
4. 视角仰角30度、方位角45度
5. 中文支持，加标题"全球机场分布3D图"
```

数据量大（5万点），建议散点 s=0.5, alpha=0.3

**作业要求：**

【必做】完成实验一全部 3 个任务，每个任务至少 2 轮 Prompt 截图，提交实验报告
