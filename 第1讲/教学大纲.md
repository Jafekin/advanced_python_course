# 第1讲 教学大纲：Python 数据处理 × Vibe Coding

## 一、本讲教学目标

### 知识目标

1. 掌握 Pandas Series 的创建、索引、统计运算及时间序列操作（shift/rolling）
2. 掌握 Pandas DataFrame 的创建、列行操作、筛选排序、分组聚合与数据重塑
3. 掌握 CSV/Excel/JSON 三种格式的读写操作，理解编码处理与缺失值基本策略
4. 掌握数据清洗核心流程：重复值检测、异常值识别（IQR/Z-score）、类型转换与描述性统计

### 能力目标

1. 能够运用两段式 Prompt 与 AI 协作完成数据处理代码的生成与迭代
2. 能够配置 Rules 规则文件规范 AI 输出风格，提升代码生成质量
3. 能够运用上下文四层级与 @文件引用为 AI 提供精准背景信息
4. 能够使用 Inline Edit + Diff 审查三步清单对 AI 生成代码进行质量把控

### 态度目标

1. 建立"AI 是编程搭档"的协作意识，形成人机协同的开发习惯
2. 培养对 AI 生成代码的批判性审查能力，不盲信、不盲从
3. 形成数据处理的工程化思维：先探索、再清洗、后分析

## 三、两条主线融合设计

| 知识主线 | Vibe Coding 工具线 | 融合方式 |
|----------|-------------------|----------|
| Series 创建与统计 | 两段式 Prompt（先氛围后约束） | 用"氛围段"描述分析目标，"约束段"指定 Series 方法和输出格式；演示一次不满意→迭代修改的完整循环 |
| DataFrame 操作链 | Rules 规则文件 | 创建 `.rules` 文件约束代码风格（变量命名、注释语言、链式调用格式），对比有/无 Rules 时 AI 输出差异 |
| 文件读写与缺失值 | 上下文四层级 + @文件引用 | 讲解四层级（系统→项目→文件→选区），演示 @引用数据文件让 AI 自动推断 dtypes 和编码方案 |
| 数据清洗与描述统计 | Inline Edit + Diff 审查三步清单 | AI 生成清洗流水线后，用 Inline Edit 局部修改异常值阈值；按三步清单（结构→逻辑→边界）审查 Diff |

## 四、重点难点与应对策略

### 重点

| 重点内容 | 教学策略 |
|----------|----------|
| Series/DataFrame 核心操作 | 对比表格展示两者异同；每个操作配合实际数据即时演示 |
| 两段式 Prompt 撰写 | 提供模板 + 反例对比，展示"好 Prompt"与"差 Prompt"的输出差异 |
| Diff 审查三步清单 | 给出真实案例，让学生找出 AI 代码中的 3 类典型问题（结构错误/逻辑遗漏/边界未处理） |

### 难点

| 难点内容 | 应对策略 |
|----------|----------|
| 时间序列 shift/rolling 的窗口概念 | 用航天进出口额数据的月度趋势做动画式演示，逐步扩大窗口观察平滑效果 |
| groupby 后的多级索引与聚合 | 拆解为"分-算-合"三步，配合可视化展示分组过程 |
| IQR 与 Z-score 的适用场景选择 | 用同一数据集分别演示两种方法，对比结果差异，总结选择依据 |
| 上下文四层级的层次关系 | 类比"国家-城市-街道-门牌号"，从最大到最小逐层演示 |

## 五、课前准备与课后作业

### 课前准备

1. **环境确认**：登录 CloudStudio，确认 Python 环境和 CodeBuddy 插件已激活
2. **数据下载**：从课程平台获取 `航天进出口额.xlsx` 数据集
3. **预习内容**：
   - 回顾 Python 基础（列表、字典、循环）
   - 阅读 Pandas 官方 10 Minutes to pandas（前半部分）
4. **工具准备**：了解 CodeBuddy 的基本交互方式（对话框、Inline Edit 入口）

### 课后作业

1. **实验完成**（课堂实验课为主，课外补完）：
   - 实验1：航天贸易数据处理全流程（4 个任务，详见实验指导书）
   - 全程使用 Vibe Coding 方式，提交 Prompt 记录与 Diff 审查截图
2. **拓展思考**：
   - 对比手动编写数据清洗代码与 AI 辅助生成的效率差异
   - 记录至少 1 次 AI 生成错误代码并通过 Diff 审查发现的案例
