# 斜杠"/"自定义指令用途

## 简单场景

一套完整的Git代码提交到远端仓库，命令比较原始，如下：

```bash
git add

git commit -m " 修复 xxx bug"

git push origin feature/deyuankong_bugfix_#12306
```

> **注**：**git指令运用可参考git基础实训教程 https://cloudstudio.net/courses/28084712534933504 ；ßß也可以询问右侧codebuddy**

这是一个高频重复的操作，我们可以通过自定义"/"指令来替代。

---

## 更多考虑

以下场景的多步操作都可以通过自定义指令进行封装，形成标准、简便的高效指令：

1. 上下文重复输入

2. 指令粒度不统一

3. 多人协作缺少统一"口令"

4. 安全与合规检查不能疏忽

5. 跨工具切换多

# 自定义"/"指令特性介绍

## 特性介绍

自定义斜杠指令（Slash Command）是在CodeBuddy中，通过固定前缀触发的快捷指令操作。

<img src='images/img1.png' width=600px>

## 特性优势

1. **快捷触发入口**：以 / 为前缀的命令更接近开发者日常的 CLI（命令行界面）体验，访问路径清晰，单一命令完成复杂任务。

2. **一致性结构化的模版**：支持事先定义上下文、约束、期望输出格式（如 Markdown、代码块、测试用例列表），统一的命令格式和参数传递方式，降低 AI 输出的随机性。

3. **可版本化、可共享**：通过仓库或配置中心托管指令，使最佳实践能够在团队内快速传播与维护。

4. **可拓展**：支持自定义项目级别、用户级别命令和实现团队协作。

5. **可调用工具执行动作**：斜杠指令不仅能生成回答，还能在固定节点调用 Skills、MCP、CI/CD流程（持续集成）触发等自动化步骤。

## 命令组成

自定义指令组成是一个 md 组成，本质都是 Prompt 提示词，和 Skills 一样存在 Frontmatter 元数据区：

```yaml
---

name: analyze # 元数据名称(可选)

description: "智能发版" # 简短描述及下述 md 正文

category: utility # 分类(可选)

complexity: basic # 复杂度级别(可选)

mcp-servers: [] # 工具调用，比如MCP 服务器(可选)

personas: [] # 角色设定，能力边界(可选)

其他：xxx # 用户自定义内容

---
```

一个完整的自定义指令一般包含以下结构，我们可以按需进行配置使用：

- **元数据** - 定义命令属性

- **标题** - 清晰的命令说明

- **触发场景** - 何时使用

- **使用方法** - 如何使用

- **执行流程** - 做什么

- **工具协调** - 用什么工具

- **关键模式** - 核心逻辑

- **示例** - 实际案例

- **边界** - 能力范围

# 快速上手

CodeBuddy 当前支持在项目级别配置和用户级别配置，在配置路径中定义 md 文档即可。其中，在 Project 项目级别配置只能在当前项目中执行，在 User 用户级别配置路径可以实现一次性配置，任意本地项目可以使用。

## 配置路径

1. **Project 项目级别配置路径**：`.codebuddy/commands`

2. **User 用户级别配置路径**：`~/.codebuddy/commands`

## 了解 CodeBuddy 的斜杠命令

在 CodeBuddy IDE 中我们内置了相关斜杠指令（Built-in Slash Commands），比如：`init`（项目初始化）、`cr`（代码审查）、`tests`（单元测试）、`explain`（代码解释）、以及 `fix`（代码修复）、`rules`（自动生成 rules 规则）、`summmarize`（总结历史对话）等指令。

在 CodeBuddy Code 中，我们面向终端工程师，提供更多丰富的操作指令，允许您在聊天中执行特殊操作、管理会话以及自定义常用工作流，更加贴合专业开发者沉浸式 Coding。
