## 一、本讲教学目标

### 1.1 知识目标

**大模型部署**：
1. 理解大模型从训练权重到可调用 API 服务的部署全景，区分推理与训练
2. 掌握主流部署方案选型：Ollama（个人/原型）、vLLM（生产/高并发）、llama.cpp（CPU/嵌入式）
3. 理解量化原理：FP16→INT8→INT4 的显存-精度-速度三角权衡，Q4_K_M 等 GGUF 量化格式
4. 理解 vLLM 核心技术：PagedAttention（KV Cache 分页管理）、连续批处理（Continuous Batching）
5. 掌握 Ollama 实操：拉取模型、起 OpenAI 兼容 API、用 openai 库改 base_url 调用

**RAG 基础与流水线**：
6. 解释 RAG 的三段式工作流：检索（Retrieve）→ 增强（Enhance）→ 生成（Generate）
7. 掌握 RAG 三代演进：Naive RAG → Advanced RAG → Modular RAG
8. 掌握 RAG 完整流水线六环节：文档加载 → 文本分块 → 向量化 → 向量库存储 → 语义检索 → Prompt 拼接生成
9. 掌握文本分块策略：固定字符分块、递归字符分块；理解重叠窗口与 chunk_size 调参
10. 掌握向量化原理：句子嵌入 vs 词嵌入、sentence-transformers 库
11. 掌握 ChromaDB 向量库：collection/document/embedding/metadata 四要素

**高级检索与记忆系统**：
12. 掌握多查询扩展 MQE：用 LLM 生成多样化查询，并行检索合并
13. 理解 HyDE（假设文档嵌入）和 Reranker（重排序）的原理与适用场景
14. 掌握 RAG 评估指标：Recall@k、Precision@k、MRR
15. 理解 Agent 记忆系统四类型：工作记忆、情景记忆、语义记忆、感知记忆
16. 理解 Memory 与 RAG 的关系：RAG 检索外部知识，Memory 检索内部交互历史

### 1.2 能力目标（Vibe Coding × 大模型应用）

1. 用 Ollama 起一个本地 OpenAI 兼容 API 服务，并用 openai 库调用
2. 用 Spec-driven 方式管理部署项目：写 deploy_speckit.md 锁定硬约束
3. 从零搭建完整 RAG 流水线：markitdown 转文档 → 递归分块 → sentence-transformers 编码 → ChromaDB 存储 → 检索 → 拼 Prompt 调 LLM
4. 用 Inline Edit 逐段调整 chunk_size、top_k 等参数，对比检索效果
5. 用 @文件引用 把知识文档注入 AI 上下文，辅助写分块函数和 Prompt 模板
6. 实现 MQE 多查询扩展，量化对比检索效果提升

### 1.3 态度目标

1. 建立"模型知识有截止日期、会幻觉"的批判意识，理解 RAG 是工程必需
2. 形成"先建知识库再问问题"的工程习惯
3. 理解"部署不只是跑起来"：要考虑显存、并发、量化等生产级问题
4. 建立"检索质量可评估"的意识：用指标量化效果，而非凭感觉

## 四、Vibe Coding 工具讲授计划

### 4.1 本讲重点

① **Spec-driven 部署规约（deploy_speckit.md）**：
- 把部署硬约束写进 constitution，每次对话附带 @deploy_speckit.md

③ **@文件引用注入知识文档**：
- 把文档路径用 @ 引用给 AI，让 AI 帮忙写分块函数和 Prompt 模板
- 对比"复制粘贴" vs "@引用"的 token 节省

### 4.2 本讲进阶

④ **两段式 Prompt 解释 RAG**：先描述需求，再逐步加约束

⑤ **Memory 作为上下文工程升级**：从"临时塞上下文"到"持久化、可检索、可遗忘"

## 五、重点难点与应对策略

| 重点/难点 | 教学策略 |
| --- | --- |
| 量化为什么能省显存且精度可接受 | "7B FP16=14GB vs INT4=4GB" + MMLU 损失<3% |
| vLLM 的 PagedAttention | 类比 OS 虚拟内存分页：KV Cache 按需分配消除碎片 |
| 为什么要分块而不是整篇向量化 | 嵌入模型 512 token 限制 + 大段稀释语义 + Prompt 长度 |
| chunk_size 和 overlap 怎么选 | 调参经验表 + Inline Edit 快速对比 |
| 句子嵌入 vs 词嵌入 | 类比：词嵌入是"单词字典"，句子嵌入是"段落指纹" |
| ChromaDB 四要素 | 类比 SQL：collection=表、document=行、embedding=索引、metadata=列 |
| HyDE 为什么有效 | 问题（疑问句）vs 文档（陈述句）语义空间不同 |
| Memory 和 RAG 的关系 | RAG=外部静态知识，Memory=内部动态交互 |

## 六、课前准备与课后作业

### 6.1 课前准备（学生）

1. 确认 CloudStudio 环境，安装依赖：
   ```bash
   pip install sentence-transformers chromadb openai markitdown pypdf
   ```
2. 安装 Ollama（本地或 CloudStudio）：
   ```bash
   curl -fsSL https://ollama.com/install.sh | sh
   ollama pull qwen2.5:7b-instruct-q4_K_M
   ```
3. 回顾第3讲的 OpenAI 兼容 API 调用方式
4. 回顾第4讲的 Transformer/注意力机制基础

### 6.2 课后作业：实验五（基于学术论文的 RAG 问答系统）

**实验数据**：`综述论文.pdf`（ACM TOSEM 2025，"LLM for Mobile: An Initial Roadmap"，北航等，29页英文综述）

| 任务 | 内容 | 知识点 | 分值 |
| --- | --- | --- | --- |
| 任务1（25分） | 文档加载与智能分块 | markitdown 转 Markdown + 递归字符分块 + 对比 chunk_size | 25 |
| 任务2（30分） | 向量化与向量库 | sentence-transformers 编码 + ChromaDB 存储 + metadata 设计 | 30 |
| 任务3（25分） | 检索与生成 | 语义检索 + Prompt 模板 + LLM 回答学术问题 | 25 |
| 任务4（20分） | 高级检索对比 | 实现 MQE 或 HyDE + Recall@5 对比评估 | 20 |

**实验要求**：
1. 从零实现，不使用 LangChain/LlamaIndex
2. 实验报告需包含检索效果对比表
3. 注意：实验数据是英文长文档，需考虑英文分块策略和跨语言查询场景

### 6.3 拓展（选做）

- 实现 Reranker 重排序，对比 Recall 提升
- 实现简化版 Memory 系统，让 RAG 助手记住历史问题
- 尝试中英文混合查询（用中文问英文文档的内容）
