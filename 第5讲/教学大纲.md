## 一、本讲教学目标

### 1.1 知识目标

1. 深入理解 Transformer 架构的核心组件：自注意力机制、多头注意力、位置编码、前馈网络
2. 理解 Encoder-Only（BERT）、Encoder-Decoder（T5）、Decoder-Only（GPT）三种预训练模型架构的异同与适用场景
3. 掌握预训练语言模型的两阶段范式：预训练（Pre-training）与微调（Fine-tuning）
4. 掌握 Hugging Face Transformers 库的核心用法：AutoTokenizer、AutoModel、pipeline、Trainer API
5. 理解全参数微调（Full Fine-tuning）的完整流程：数据集构建 → Tokenization → 训练循环 → 评估
6. 了解参数高效微调（PEFT）的动机与核心思想：LoRA 低秩适配原理
7. 理解训练过程中的关键超参数：学习率、batch size、epoch、warmup、weight decay

### 1.2 能力目标

1. 能使用 `pipeline` 快速调用 BERT 完成文本分类、NER、问答等任务
2. 能使用 `AutoTokenizer` + `AutoModel` 完成自定义推理
3. 能基于 hotel.csv 数据集对 BERT 进行情感分类微调，跑通完整训练流程
4. 能理解并分析训练 loss 曲线，判断欠拟合/过拟合
5. 能将 Vibe Coding 工具（CodeBuddy + Speckit）应用于模型微调代码的编写与调试

### 1.3 态度目标

1. 建立"预训练 + 微调"是现代 NLP 工程的主流范式的认知
2. 理解为什么不需要从头训练模型，迁移学习的价值
3. 形成"看懂架构 → 用库调用 → 再理解原理"的工程化学习路径

## 三、知识线与工具线融合设计

**【核心设计思路】** 本讲是"从调用者到理解者"的深化：第3讲学生已能通过 API 调用大模型，本讲让他们理解模型内部架构，并亲手微调一个真实的 BERT 模型，完成从"黑盒调用"到"白盒定制"的跨越。

| **知识点** | **与前几讲的关联** | **融合方式** |
| --- | --- | --- |
| Transformer 自注意力 | 第3讲提到的 Decoder-Only 架构 | 补全 Encoder 侧的机制，形成完整图景 |
| BERT 掩码语言模型 | 第3讲的 Token/BPE 概念 | MLM 是在 Token 级别上做的预测任务 |
| Hugging Face pipeline | 第3讲的 OpenAI API 调用 | 两者都是"输入文本→输出结果"，但一个本地一个云端 |
| 全参数微调训练循环 | 第2讲的 sklearn `model.fit()` | 都是"数据→损失→反向传播→更新参数"，只是框架不同 |
| LoRA 低秩适配 | 第2讲的 PCA 降维思想 | 都是用低维空间近似高维变化 |
| Speckit + CodeBuddy | 贯穿全课的 Vibe Coding 工具线 | 用 Speckit 描述微调任务，让 CodeBuddy 生成训练代码框架 |

## 四、重点难点与应对策略

| **重点/难点** | **教学策略** |
| --- | --- |
| 难点：自注意力机制的 Q/K/V 矩阵 | 用"图书馆查书"类比：Q=你的问题，K=书名标签，V=书的内容；手动推导3×3小例子 |
| 重点：BERT vs GPT 架构差异 | 对比表格 + 任务适配图：BERT 适合理解类任务，GPT 适合生成类任务 |
| 难点：微调训练循环的代码结构 | 提供完整注释的教学骨架，先理解流程再补全细节 |
| 重点：Tokenizer 的 padding/truncation | 用酒店评论数据演示不同长度文本的处理方式，对比 `input_ids` 形状 |
| 难点：LoRA 的低秩分解原理 | 用 PCA 类比：原始权重矩阵变化 ΔW = AB，A 和 B 是"压缩版"的更新 |
| 重点：超参数调优直觉 | 展示不同学习率/epoch 的 loss 曲线，建立"曲线形状→诊断结论"的直觉 |

**思考题（三选一）**：

a. 为什么 BERT 使用 Masked Language Model（MLM）而不是 Causal Language Model（CLM）进行预训练？MLM 和 CLM 的训练目标分别适合哪类下游任务？

b. 在微调 BERT 时，学习率通常设为 2e-5，远小于训练 sklearn 模型时的默认值。为什么预训练模型的微调需要更小的学习率？如果设置过大会发生什么？

c. LoRA 的核心思想是将权重更新 ΔW 分解为低秩矩阵乘积 AB。为什么这个假设在微调场景下是合理的？它和 PCA 的降维思想有什么相似之处？
