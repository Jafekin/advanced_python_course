### Slide 5｜什么是机器学习？
**传统编程思路**：
```
规则 + 数据 → 答案
```
人为定义规则（如：垃圾邮件 = 包含"中奖"、"免费"等词）

**机器学习思路**：
```
数据 + 答案 → 规则（模型）
```
从标注数据中学习规则

> **核心区别**：机器学习从数据中自动发现模式，而非人工编写规则

```
┌─────────┐
│  数据    │ ← AI 的"燃料"
├─────────┤
│  算法    │ ← 模型如何学习
├─────────┤
```

- **数据**：样本 + 标签（监督学习）
- **算法**：线性回归、SVM、决策树等

```
数据 → 特征工程 → 模型训练 → 模型评估 → 服务部署
              ↑___________________________↓
                    迭代优化
```

1. **获取数据**：样本 + 标签

6. **迭代优化**：调整超参数、增加数据

## Part 1：线性回归（监督学习 - 回归）

### Slide 12｜回归任务：预测连续数值
**回归**：输入特征 X → 预测连续数值 y

**应用场景**：
- 房价预测（面积、位置 → 价格）
- 销量预测（广告投入 → 销量）
- 成绩预测（学习时间 → 成绩）

**本讲任务**：学习时间 → 学生成绩（data.xlsx）

---

### Slide 13｜简单线性回归
**模型**：y = ax + b

- x：输入特征（学习时间）
- y：输出（成绩）
- a：回归系数（斜率）
- b：截距

**目标**：找到最佳 a、b，使得预测值 ŷ 尽可能接近真实值 y

---

### Slide 14｜损失函数（Loss Function）
**衡量模型预测与真实值的差距**

**均方误差（MSE）**：
```
MSE = (1/m) × Σ(y⁽ⁱ⁾ - ŷ⁽ⁱ⁾)²
```

- m：样本数
- y⁽ⁱ⁾：第 i 个样本的真实值
- ŷ⁽ⁱ⁾：第 i 个样本的预测值

**目标**：最小化 MSE

---

### Slide 15｜梯度下降法
**优化算法**：通过迭代找到损失函数的最小值

```
θ = θ - η × ∇L(θ)
```

- θ：模型参数（a, b）
- η：学习率（步长）
- ∇L(θ)：损失函数的梯度

**过程**：
1. 随机初始化 θ
2. 计算梯度 ∇L(θ)
3. 更新 θ
4. 重复 2-3 直到收敛

---

### Slide 16｜学习率的影响
| 学习率 η | 效果 |
|---|---|
| 过小 | 收敛慢，需要更多迭代 |
| 合适 | 稳定收敛到最小值 |
| 过大 | 震荡甚至发散，无法收敛 |

> 学习率是超参数，需要通过实验调整

---

### Slide 17｜多元线性回归
**模型**：y = w₀ + w₁x₁ + w₂x₂ + ... + wₙxₙ

- 多个特征 x₁, x₂, ..., xₙ
- 多个系数 w₁, w₂, ..., wₙ
- 截距 w₀

**矩阵表示**：ŷ = X_b · w

- X_b：增广特征矩阵（加一列 1）
- w：参数向量

---

### Slide 18｜正规方程解
**直接求解最优参数**：
```
w = (X_b^T · X_b)^(-1) · X_b^T · y
```

**优点**：
- 不需要迭代
- 不需要学习率

**缺点**：
- 时间复杂度 O(n³)
- 大数据集计算慢

> sklearn 的 LinearRegression 默认使用正规方程

---

### Slide 19｜回归评估指标
| 指标 | 公式 | 说明 |
|---|---|---|
| **MSE** | (1/m) × Σ(y - ŷ)² | 均方误差，量纲变化 |
| **RMSE** | √MSE | 均方根误差，与 y 同量纲 |
| **MAE** | (1/m) × Σ|y - ŷ| | 平均绝对误差 |
| **R²** | 1 - MSE/Var(y) | 判定系数，[-∞, 1] |

**R² 解释**：
- R² = 1：完美预测
- R² = 0：和猜均值一样
- R² < 0：比猜均值还差

---

### Slide 20｜过拟合与欠拟合
| 现象 | 训练误差 | 测试误差 | 原因 | 解决 |
|---|---|---|---|---|
| **欠拟合** | 高 | 高 | 模型太简单 | 增加复杂度、更多特征 |
| **恰好** | 低 | 低 | 模型合适 | — |
| **过拟合** | 低 | 高 | 模型太复杂 | 降低复杂度、正则化、增加数据 |

> 模型训练的目的是预测未知数据，而不是拟合所有已知数据

## Part 2：SVM 分类（监督学习 - 分类）

### Slide 22｜分类任务：预测离散类别
**分类**：输入特征 X → 预测离散类别 y

**应用场景**：
- 垃圾邮件检测（正常/垃圾）
- 疾病诊断（良性/恶性）
- 手写数字识别（0-9）

**本讲任务**：鸢尾花分类（3类）

---

### Slide 23｜KNN 算法（K 近邻）
**核心思想**：物以类聚，近朱者赤

**算法步骤**：
1. 计算测试样本与所有训练样本的距离
2. 选择距离最近的 K 个样本
3. 这 K 个样本中多数属于哪类，测试样本就属于哪类

**超参数**：
- K：邻居数量
- 距离度量：欧氏距离、曼哈顿距离等

---

### Slide 24｜逻辑回归
**虽然叫"回归"，但用于分类**

**核心思想**：
1. 线性回归：ŷ = w^T · x
2. Sigmoid 函数：σ(t) = 1 / (1 + e^(-t))
3. 逻辑回归：ŷ = σ(w^T · x) ∈ [0, 1]

**决策规则**：
- ŷ ≥ 0.5 → 类别 1
- ŷ < 0.5 → 类别 0

---

### Slide 25｜支持向量机（SVM）
**核心思想**：找到最大间隔的超平面

```
         支持向量
            ↓
    ○ ○    |    ● ●
      ○    |    ●
           |
  ─────────┼───────── 超平面
           |
      ●    |    ○
    ● ●    |    ○ ○
            ↑
         支持向量
```

- **超平面**：分类决策边界
- **支持向量**：距离超平面最近的样本点
- **间隔**：支持向量到超平面的距离

**目标**：最大化间隔

---

### Slide 26｜SVM 的核函数
**核函数**：将数据映射到高维空间，使线性不可分变为线性可分

| 核函数 | 公式 | 适用场景 |
|---|---|---|
| **线性核** | K(x, x') = x^T · x' | 线性可分 |
| **多项式核** | K(x, x') = (ax^T · x' + b)^d | 非线性 |
| **高斯核（RBF）** | K(x, x') = exp(-γ||x-x'||²) | 最常用，映射到无穷维 |

> 高斯核（RBF）是最常用的核函数，适合大多数非线性数据

---

### Slide 27｜Hard Margin vs Soft Margin SVM
**Hard Margin SVM**：
- 所有样本必须正确分类
- 对异常点敏感
- 容易过拟合

**Soft Margin SVM**：
- 允许少量样本分类错误
- 引入松弛变量 ξ
- 超参数 C 控制容错程度

**C 的影响**：
- C 大：间隔小，错误少，易过拟合
- C 小：间隔大，错误多，易欠拟合

---

### Slide 28｜分类评估：混淆矩阵
| | 预测为正 | 预测为负 |
|---|---|---|
| **实际为正** | TP（真阳性）| FN（假阴性）|
| **实际为负** | FP（假阳性）| TN（真阴性）|

**示例**：癌症预测
- TP：实际患癌，预测患癌（正确）
- FN：实际患癌，预测健康（漏诊）
- FP：实际健康，预测患癌（误诊）
- TN：实际健康，预测健康（正确）

---

### Slide 29｜精确率、召回率、F1-score
| 指标 | 公式 | 含义 | 关注场景 |
|---|---|---|---|
| **精确率 P** | TP / (TP + FP) | 预测为正中，真正正例的比例 | 宁可漏诊，不可误诊 |
| **召回率 R** | TP / (TP + FN) | 实际为正中，被找到的比例 | 宁可误诊，不可漏诊 |
| **F1-score** | 2PR / (P + R) | P 和 R 的调和平均 | 兼顾两者 |

**示例**：癌症预测
- 精确率 40%：预测患癌的 10 人中，4 人真的患癌
- 召回率 80%：10 个癌症患者中，8 人被找到

---

### Slide 30｜准确率的陷阱
**场景**：癌症预测系统，准确率 99.9%

| | 预测患癌 | 预测健康 |
|---|---|---|
| 实际患癌（10人）| 0 | 10 |
| 实际健康（9990人）| 0 | 9990 |

- **准确率**：9990/10000 = 99.9% ← 看起来很高
- **召回率**：0/10 = 0% ← 一个癌症患者都没找到！

> **结论**：准确率高 ≠ 模型好。必须看 P/R/F1

## Part 3：K-means 聚类（无监督学习）

### Slide 32｜聚类任务：发现数据分组
**聚类**：输入特征 X → 发现数据分组（无标签）

**应用场景**：
- 客户分群（高消费/低消费）
- 异常检测（离群点）
- 图像分割（相似像素分组）

**本讲任务**：购物中心客户分群（mall_customers.csv）

---

### Slide 33｜K-means 算法原理
**目标**：将 n 个样本划分为 k 个簇

**算法步骤**：
1. 随机初始化 k 个聚类中心
2. 将每个样本分配到最近的聚类中心
3. 重新计算每个簇的中心（样本均值）
4. 重复 2-3 直到中心不再变化

**损失函数**：
```
J = Σᵢ Σⱼ ||x⁽ⁱ⁾ - μⱼ||²
```
- μⱼ：第 j 个簇的中心
- 目标：最小化 J（各样本到中心的距离平方和）

---

### Slide 34｜K-means 的迭代过程
```
初始化          第1次迭代        第2次迭代        收敛
  ○ ○            ○ ○              ○ ○              ○ ○
    ●              ●                ●                ●
  ×   ×          ×   ×            ×   ×            ×   ×

  ○ ○            ○ ○              ○ ○              ○ ○
    ●              ●                ●                ●
  ×   ×          ×   ×            ×   ×            ×   ×
```

- ○：簇 1 的样本
- ●：簇 2 的样本
- ×：聚类中心

> 通常 5-10 次迭代即可收敛

---

### Slide 35｜肘部法则确定 K
**问题**：K 取多少合适？

**肘部法则**：
- 绘制 loss（inertia_）随 K 变化的曲线
- 找"拐点最明显"处作为 K 值

```
loss
  │  ●
  │   ●
  │    ●
  │     ● ← 拐点
  │      ●
  │       ●
  │        ●
  └──────────── K
    2 3 4 5 6 7 8 9
```

> loss 下降突然变缓的点 = 最佳 K

---

### Slide 36｜K-means 的优缺点
**优点**：
- 原理简单，易于实现
- 收敛速度快
- 可解释性强

**缺点**：
- 需要预先指定 K
- 对初始中心敏感
- 对异常点敏感
- 只能发现球形簇

**改进方法**：
- K-means++：改进初始化
- 多次运行取最优
- 肘部法则选 K

## Part 4：PCA 降维（无监督学习）

### Slide 38｜降维任务：压缩特征维度
**降维**：高维特征 → 低维特征（无标签）

**目的**：
- 数据可视化（高维 → 2D/3D）
- 减少计算量
- 去除噪声
- 保留关键信息

**本讲任务**：鸢尾花 4 维 → 2 维可视化

---

### Slide 39｜PCA 原理（主成分分析）
**核心思想**：找到方差最大的方向

**步骤**：
1. 去中心化（坐标原点移到数据中心）
2. 找方差最大方向 = 第一主成分
3. 与第一主成分正交且方差次大 = 第二主成分
4. 以此类推

**降维标准**：信息损失最小，即方差最大

---

### Slide 40｜PCA 的数学原理
**协方差矩阵**：
```
C = (1/m) X^T X
```

**特征值分解**：
```
C v = λ v
```

- v：特征向量（主成分方向）
- λ：特征值（该方向的方差）

**选择前 k 个主成分**：
- 按特征值从大到小排序
- 选择前 k 个特征向量
- 投影到低维空间

---

### Slide 41｜PCA 为什么要先标准化？
**不标准化的后果**：
- 量纲大的特征（如年收入 20000）主导主成分方向
- 量纲小的特征（如年龄 30）被忽略

**标准化**：
```
x_scaled = (x - μ) / σ
```

- μ：均值
- σ：标准差

> PCA 前必须 StandardScaler

---

### Slide 42｜解释方差比
**explained_variance_ratio_**：
- 每个主成分解释的方差比例
- 累计解释方差 = 保留的信息量

**示例**：
```
主成分 1：60%
主成分 2：30%
主成分 3：8%
主成分 4：2%
```

> 前 2 个主成分解释了 90% 的信息，降到 2 维损失 10%

---

### Slide 43｜PCA 降维的维度选择
**方法 1**：累计解释方差达到阈值（如 95%）

```python
pca = PCA(0.95)  # 保留 95% 信息
pca.fit(X)
print(pca.n_components_)  # 自动计算需要的维度
```

**方法 2**：绘制累计方差曲线，找拐点

> 通常选择累计方差达到 85%-95% 的维度

---

### Slide 44｜PCA 的优缺点
**优点**：
- 无参数限制（不受局部最优影响）
- 可消除数据相关性
- 提高算法计算效率
- 数据可视化

**缺点**：
- 可能丢失有用信息
- 主成分解释性差
- 只适用于线性降维

> 某些情况下，PCA 降维后分类性能反而提升（降噪效果）

## Part 5：Spec-driven 开发范式与 ML 项目规约化

### Slide 46｜从 Vibe Coding 到 Spec-driven

**Vibe Coding 的局限**：
- 单次对话完成单个任务没问题
- 多环节项目（数据→建模→评估→可视化）容易"前后矛盾"
- AI 上下文窗口有限，复杂任务丢失约束

**Spec-driven 的核心思路**：
- 先定义"做什么、不能做什么、验收标准"
- 再让 AI 按规约逐步执行
- 维护软件 = 维护规约，而非维护代码

> Speckit 是 AI 时代的工程化"护栏系统"

---

### Slide 47｜什么是 Speckit？

**GitHub 开源工具**（111k+ stars）：把 AI 编程从"一句话生成代码"升级为"先写规约，再按规约执行"

**核心工作流**：
```
constitution → specify → plan → tasks → implement
  (底线原则)    (需求)    (方案)   (拆解)    (落地)
```

**安装方式**：
```bash
uv tool install specify-cli --from git+https://github.com/github/spec-kit.git
```

> 当前 CodeBuddy IDE 和 CodeBuddy Code 均已支持 Speckit 斜杠指令

---

### Slide 48｜Speckit 核心命令

| 命令 | 角色 | 作用 | 产出 |
|---|---|---|---|
| `/speckit.constitution` | 项目创始人 | 定义不可协商原则 | `constitution.md` |
| `/speckit.specify` | 产品经理 | 用业务语言写需求 | `spec.md` |
| `/speckit.clarify` | 测试人员 | 对模糊点提问补齐 | 更新后的 `spec.md` |
| `/speckit.plan` | 架构师 | 生成技术方案 | `plan.md` |
| `/speckit.tasks` | 项目经理 | 任务拆解与排序 | `tasks.md` |
| `/speckit.analyze` | 审查者 | 检查一致性 | 分析报告 |
| `/speckit.implement` | 程序员 | 按任务落地代码 | 可运行代码 |

---

### Slide 49｜ML 项目中的 Speckit 实践

**ML 场景下 constitution 应包含的原则**：
```markdown
# ML 项目宪法
- 禁止在全量数据上 fit StandardScaler（防数据泄露）
- 聚类前必须用肘部法则确定 K，不得凭感觉指定
- 所有随机过程必须设置 random_state=42
- PCA 前必须先 StandardScaler
- 评估分类模型必须输出 F1-score，禁止只看 accuracy
```

**speckit.md 结构**：
```markdown
## constitution（不可违反的硬约束）
## context（数据背景 + 分析目标）
## 任务列表（可执行、可验证的步骤）
```

---

### Slide 50｜Speckit 简化版用法：@speckit.md 引用

**适合课程实验的轻量做法**：
1. 手写 `speckit.md`（constitution + context + 任务列表）
2. 每次对话 Prompt 附带 `@speckit.md` 引用
3. AI 自动遵守 constitution 中的约束

**示例**：
```
@speckit.md @mall_customers.csv 
用肘部法则确定 K：K 从 2 到 9，绘制 inertia 曲线
```

> 核心收益：约束不用每次重复写，AI 全程遵守 constitution

### Slide 51｜Speckit vs 两段式 Prompt 对比

| | 两段式 Prompt | Speckit |
|---|---|---|
| **适合** | 单任务、一次性建模 | 多环节项目、跨对话保持约束 |
| **优点** | 简单快速 | 约束不重复，AI 全程遵守 constitution |
| **缺点** | 新开对话约束断掉 | 需要额外写 speckit.md |
| **ML 场景** | 单算法快速验证 | 完整建模项目（探查→建模→评估→可视化）|

> 两者不冲突：Speckit 管全局约束，两段式 Prompt 管单步执行

### Slide 54｜sklearn 常用 API 速查

| 功能 | API | 说明 |
|---|---|---|
| 线性回归 | `LinearRegression().fit(X, y)` | X 必须二维 |
| 回归系数 | `model.coef_` / `model.intercept_` | 系数/截距 |
| RMSE | `np.sqrt(mean_squared_error(y, y_pred))` | 开方 MSE |
| SVM 分类 | `svm.SVC(kernel='rbf', C=100)` | rbf 核最常用 |
| F1-score | `f1_score(y_pred, y_test, average='micro')` | 多分类用 micro/macro |
| 数据划分 | `train_test_split(X, y, test_size=0.1)` | 必设 random_state |
| K-means | `KMeans(n_clusters=k, init='k-means++')` | 无监督，只有 X |
| 肘部法则 | `model.inertia_` | 距离平方和 |
| PCA | `PCA(n_components=2).fit_transform(X)` | 先标准化 |
| 标准化 | `StandardScaler().fit_transform(X)` | PCA/SVM/KNN 必备 |

### Slide 55｜Vibe Coding 建模关键词速查

| 术语 | 含义 |
|---|---|
| 两段式 Prompt（建模）| Round 1 描述任务类型和数据；Round 2 指定算法、指标、格式 |
| Speckit / speckit.md | 规约文件，包含 constitution + context + 任务列表，通过 @引用注入对话 |
| 数据泄露 | 在全量数据或测试集上 fit Scaler，导致评估失真 |
| reshape(-1, 1) | 把一维数组转为列向量，sklearn 要求 X 是二维矩阵 |
| random_state | 随机种子，确保结果可复现（Rules 规定必须设置）|
| 肘部法则 | 通过 loss 随 K 变化曲线找拐点，确定 K-means 的 K 值 |
| 决策边界 | 分类模型在特征空间中的分类分界线 |
| 主成分 | PCA 中方差最大的方向，前 k 个主成分构成低维表示 |
| 监督/无监督 | 有标签（分类/回归）vs 无标签（聚类/降维）|
| constitution | Speckit 中的"宪法"，定义不可违反的项目原则 |
